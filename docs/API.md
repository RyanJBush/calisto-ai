# API Reference (Local Demo)

Base URL: `http://localhost:8000`

Swagger: `http://localhost:8000/docs`

## Important behavior note

`POST /api/chat/query` returns `answer_mode: grounded_heuristic` in the current implementation. That means the response is composed heuristically from retrieved chunks (with citations), not generated by a live LLM provider.

## Core endpoints

### Auth
- `POST /api/auth/register`
- `POST /api/auth/login`

### Documents
- `POST /api/documents/upload`
- `GET /api/documents`
- `GET /api/documents/{document_id}`
- `POST /api/documents/collections`
- `POST /api/documents/{document_id}/access`

### Chat
- `POST /api/chat/query`
- `GET /api/chat/history`
- `POST /api/chat/feedback`

### Admin
- `GET /api/admin/metrics`

### Health
- `GET /health`

## Example: login

```bash
curl -X POST http://localhost:8000/api/auth/login \
  -H 'Content-Type: application/json' \
  -d '{"email":"admin@calisto.ai","password":"password123"}'
```

## Example: grounded query

```bash
curl -X POST http://localhost:8000/api/chat/query \
  -H "Authorization: Bearer $TOKEN" \
  -H 'Content-Type: application/json' \
  -d '{"query":"How many PTO days do employees get?","top_k":3,"grounded_mode":true}'
```

Expected fields to watch:
- `answer`
- `answer_mode` (`grounded_heuristic`)
- `citations`
- `confidence_score`
- `insufficient_evidence`
- `latency_breakdown_ms`
